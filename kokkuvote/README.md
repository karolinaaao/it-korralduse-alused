# Kokkuvõte

Git on versioonihaldussüsteem, mis aitab jälgida muudatusi failides ja teha koostööd.  
Seda kasutatakse selleks, et töö ei läheks kaduma ja kõik muudatused oleksid dokumenteeritud.  
Commit on muudatuse salvestamine lokaalselt, push saadab need GitHubi.  
Harud ehk branches on kasulikud, sest need võimaldavad teha muudatusi ilma põhiharule mõju avaldamata.  
Pull Request on viis, kuidas üks haru ühendada teisega ja lasta kellelgi muudatused üle vaadata.  
Git erineb failide kopeerimisest USB-le, sest Git salvestab kogu ajaloo ja muudatused.  
Issues on nagu ülesanded, mis aitavad projekti planeerida ja jälgida.  
Kanban tahvel GitHubis aitab näha töövoogu ja prioriteete.  
GitHubi kasutaksin edaspidi kindlasti, sest see teeb projektide haldamise lihtsamaks.  
See töö aitas mul mõista, kuidas Git ja GitHub päriselt toimivad.  
Kõige raskem oli alguses aru saada harudest ja PR-idest.  
Kõige lihtsam oli failide loomine ja README-de täitmine.  
Sain aru, kui oluline on commitide selge sõnastus.  
Õppisin, kuidas struktureerida repositooriumi ja hoida sisu korras.  
Kokkuvõttes oli see praktiline töö väga kasulik ja arendav.
